# AMS v1.3 Correction Capture Controller Plan

Date: 2026-05-28

Status: implementation-verified

## Goal

Make AMS intercept live behavioral drift, not only validate memories after a trace is complete.

AMS remains centered on the thesis:

```text
memory is verified experience that improves future action
```

The Correction Capture Controller adds the missing live loop:

```text
user correction -> stop lane -> classify mistake -> record affected files/actions -> route to memory/ledger/gate -> require explicit resume
```

## Architecture Position

The corrected AMS shape is:

```text
Memory Use Controller
  - startup brief
  - scoped retrieval
  - required directive checks

Correction Capture Controller
  - live correction detection
  - mistake classification
  - affected-action recorder
  - directive/CEM/ledger router
  - resume gate

CEM Validation Kernel
  - trace ingest
  - atom proposal
  - quarantine
  - promotion
  - action brief

Monitor
  - required directives
  - memory health
  - correction-loop health
  - drift checks
```

## V0 Detection Scope

The deterministic V0 classifier detects these correction categories:

- `workflow_violation`
- `premature_implementation`
- `scope_trimming`
- `source_skipping`
- `generic_framing`
- `memory_miss`
- `fake_completion`
- `repeated_drift`
- `approval_gate_violation`

Example triggers include:

- "I shouldn't have to tell you this"
- "we already said no"
- "you trimmed it again"
- "why are you building before planning"
- "you didn't read the source"
- "this is half-baked"
- "save this to memory"
- "you claimed completion without verification"

## Event Schema

Each correction event records:

- `event_id`
- `detected_at`
- `source`
- `user_text`
- `mistake`
- `categories`
- `affected_files`
- `affected_actions`
- `stale_memory_ids`
- `route_targets`
- `routes`
- `resume_required`
- `resume_status`
- `resume_token`
- `domain_scope`
- `task_family`
- `session_id`

Records are written under the AMS root:

```text
correction-events.jsonl
correction-latest.json
correction-resume-gate.json
correction-resume-runs.jsonl
```

## Routing

On capture, the controller routes the correction to:

- `ams_directive`: pin a future-behavior directive when the correction has reusable behavioral force.
- `cem_candidate_experience_atom`: save a failure-mode experience through the normal CEM validation path.
- `project_ledger_entry`: append a project ledger entry when a ledger path is supplied.
- `stale_or_contradicted_memory`: link stale/contradicted memory ids or mark review pending.
- `human_approval_gate`: block continuation until explicit resume approval.

## CLI

Capture a correction:

```powershell
python scripts/ams.py correction capture "why are you building before planning" --affected-file package.json --affected-action "created files before plan approval"
```

Inspect the gate:

```powershell
python scripts/ams.py correction gate
```

Resume after approval:

```powershell
python scripts/ams.py correction resume <event_id> --approved-by Hammad
```

List events:

```powershell
python scripts/ams.py correction list
```

## Monitor Checks

Monitor-0 now checks:

- Correction Capture Controller wiring exists.
- The correction resume gate is clear.
- Correction event records are visible when present.
- The correction-capture directive surfaces in action briefs.

An active correction gate makes Monitor-0 fail visibly until `correction resume` clears it.

## Acceptance Tests

- User says "I shouldn't have to tell you this" and a correction is classified.
- User says "we already said no X" and stale/contradicted memory routing is activated.
- Agent violates a plan-first gate and affected files/actions are recorded.
- Correction becomes both an explicit directive route and a CEM failure-mode candidate.
- Action brief retrieves the correction behavior later.
- Active correction gate blocks monitor health until explicit resume.

## Non-Scope

- no always-on transcript tap yet;
- no IDE/browser runtime hook yet;
- no automatic edits to arbitrary project ledgers unless a ledger path is supplied;
- no replacement for CEM validation;
- no Vol ownership of correction capture.
