**Recommendation:** choose **LanceDB-Node**. **Rejected alternatives:** **SQLite + sqlite-vec** is attractive for simple local metadata, but it creates a second vector implementation and does not directly read SC’s LanceDB store. **DuckDB + DuckDB-VSS** has good analytical ergonomics but is a different operational substrate and still requires SC read translation. **Local Postgres + pgvector** is the most mature relational choice but violates the “no separate process if possible” constraint and is overkill for a single-host personal memory store.
